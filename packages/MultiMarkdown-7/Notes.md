# Testing #

Periodically, and before each version bump:


* Integration testing with ctest

	This includes the `Emphasis Test` and `Reference Test` which are generated by code.  I should develop more exhaustive tests like this.


* Perform memory testing with valgrind

		valgrind --leak-check=full --track-origins=yes --show-leak-kinds=all -s  ./multimarkdown batch -r -t html  ../tests/MMD7Tests/*.text
		valgrind --leak-check=full --track-origins=yes --show-leak-kinds=all -s  ./multimarkdown batch -r -t html -c -x .htmlc ../tests/MMD7Tests/*.text
		valgrind --leak-check=full --track-origins=yes --show-leak-kinds=all -s  ./multimarkdown batch -r -t latex -c  ../tests/MMD7Tests/*.text
		valgrind --leak-check=full --track-origins=yes --show-leak-kinds=all -s  ./multimarkdown batch -r -t latex  ../tests/MMD7Tests/*.text


* Performance testing in /dev

	MMD should be faster than (or equal to) CommonMark in almost all cases
	(might need to look at compatibility mode specifically for some tests).

	Deep blockquote is slow -- this shouldn't be a problem in real life, but
	could be a problem with pathologic input (e.g. >>>>....).  I need to
	look into this

		make run
		make clean


* Fuzz testing in /fuzz

	Need to look for at least easily discovered bugs.  And periodically should let it run for a long time.

		make
		cd build
		make
		./fuzz_mmd-7

	Can also fuzz test over test suite to catch leaks and errors:

		./fuzz_mmd-7 ../../tests/MMD7Tests/*.text
